## Scope and interpretation

This inventory answers a narrower question than a disk-usage report: which
GEODE artifacts can become dialogue, tool-use, search, or decision
trajectories? It separates data already public in this repository from local
sources that still require review.

Counts below are audit measurements, not a promise that every physical file is
an independent training example. Content hashes and normalized event hashes
were used where available. Sources and subsets overlap, so totals must not be
added across rows without the stated de-duplication key.

The availability labels are:

- `published` — public now, within the stated fidelity;
- `review_required` — useful candidate, but not cleared for publication;
- `restricted` — allowlisted private export only;
- `excluded` — not eligible for trajectory publication.

No private payload, workstation username, credential, or absolute source path
is included in this inventory.

## Exclusions

- Raw reasoning diagnostics and reasoning sidecars: `excluded` because hidden
  reasoning is not required for reproducibility and can contain sensitive
  model internals.
- General private MCP stores: `excluded` from bulk export because they contain
  account and third-party data unrelated to a bounded evaluation.
- Opaque checkpoint blobs: `restricted` or `excluded` until decoded through a
  field-aware, redaction-aware extractor.
- The checkpoint retirement receipt itself: class
  `retirement_receipt`, not a trajectory and not a backup.

## Recommended publication order

1. Use the already-public tau2 and Petri data as the first normalized dated
   release; publish only manifests and normalized views, leaving raw artifacts
   immutable.
2. Join and redact MCPMark rich sessions against the 89 public verifier
   artifacts, retaining completeness and failure provenance.
3. Normalize autoresearch as search trajectories with campaign-lineage splits.
4. Review the 152 additional Petri logs for licensing and payload safety.
5. Export runtime sessions only from an explicit evaluation-task allowlist.

No local candidate should be deleted merely because it appears in this
inventory. The retirement gate in [`TRAJECTORIES.md`](../../TRAJECTORIES.md)
requires a validated destination or a documented exclusion decision first.
